# OCR Auto Policy v1 — Calibration Harness

This document describes how to run the evidence-only evaluation harness for
Auto Policy v1 (shipped in PR 100). The harness makes **no production code
changes** and does **not** alter the 3-page floor or 30% fraction thresholds.
Its output is telemetry only; any policy change requires a separate PR with
empirical evidence from this harness.

## Prerequisites

* Windows or Linux workstation.
* Python 3.11+ with the project dev extras installed
  (`pip install -e .[dev]`).
* PyMuPDF and Pillow (used by the synthetic PDF generator; test extras
  already include these).
* Tesseract on `PATH` (for the `tesseract` treatment).
* NVIDIA GPU (RTX 3060 or better) with the Unlimited-OCR model installed
  for the `unlimited_ocr` and `auto` treatments::

    aksharamd models install unlimited_ocr

  Without the model, the harness will still run to completion — the UOC
  and auto treatments will exit with a "model not installed" fallback that
  the review queue flags as `treatment_failed`.
* `nvidia-smi` on `PATH` if you want VRAM sampling. Optional; the harness
  falls back to `peak_vram_mib=None` when the tool is absent.

## Corpus

The corpus is 20 documents (or fewer, if some are not yet fetched):

* **12 ParseBench PDFs** listed in `benchmarks/parsebench_assets.lock.json`.
  These are reference-fetch-only; run `benchmarks/parsebench_fetch.py` with
  `AKSHARAMD_PARSEBENCH_ALLOW_NETWORK=1` to populate the local cache.
* **8 synthetic profile PDFs** generated by
  `python -m benchmarks.ocr_auto_calibration.synthetics`. Idempotent — the
  generator only rewrites files whose recipe hash changed.
* **0–2 failure fixtures** (real-first priority). A canonical
  cropped-rotated-scaled hallucination case is not yet catalogued; when one
  is confirmed, drop it under
  `benchmarks/ocr_auto_calibration/fixtures/failure/` with a sibling JSON
  label. Until then, only a deterministic zero-render fixture is used (and
  only if one is added — none is committed by default).

## Generating the synthetic fixtures

```powershell
python -m benchmarks.ocr_auto_calibration.synthetics
```

Output is written under `benchmarks/ocr_auto_calibration/fixtures/synthetic/`.
The generator prints one line per profile stating whether the fixture was
regenerated or was already up to date.

## Running the harness

Full 20-document pass with caching enabled (recommended):

```powershell
python -m benchmarks.ocr_auto_calibration.run --resume
```

Options:

* `--dry-run` — mock every compile invocation. Produces schema-conformant
  results with fabricated numbers so you can validate the report layout
  without a GPU.
* `--corpus parsebench|synthetic|failure|all` — restrict the run to a
  single corpus source (default: `all`).
* `--treatments tesseract,unlimited_ocr,auto` — restrict to a subset of the
  three treatments.
* `--output-dir <path>` — override the default output location
  (`benchmarks/ocr_auto_calibration/results/<date>_schema_v<version>/`).
* `--resume` — honour the on-disk cache. Without this flag the cache is
  ignored and every treatment is re-executed.

### Runtime expectations

Rough numbers on an RTX 3060 (12 GB VRAM):

* Tesseract treatments — a few minutes total for all 20 docs.
* Unlimited-OCR treatments — GeoTopo alone can take ~70 minutes; expect
  ~3–6 hours for the full 20-document set with UOC.
* Auto treatments — similar to Unlimited-OCR when the policy chooses UOC,
  similar to Tesseract when it chooses Tesseract.

## Artifacts produced

Under the output directory:

* `run_report.json` — the full :class:`RunReport` with one entry per document
  (Tesseract / UOC / auto).
* `OCR_AUTO_CALIBRATION_<date>.md` — human-readable summary with the
  confusion table and per-document detail.
* `review_queue.json` — every case flagged for human review with priority
  bucket and reason codes.

Under the harness directory:

* `benchmarks/ocr_auto_calibration/.cache/` — content-addressable cache
  keyed on `(document_sha256, treatment, aksharamd_commit, model_revision,
  harness_schema_version)`. Safe to delete at any time; the next run will
  rebuild it. Any commit / model / schema change naturally invalidates all
  entries.
* `benchmarks/ocr_auto_calibration/.compile_outputs/` — the compile outputs
  for each `(document_id, treatment)` pair. Kept so a reviewer can inspect
  the actual Markdown produced.

## Interpreting the review queue

Each entry has the shape::

    {
      "doc_id": "...",
      "treatment": "tesseract | unlimited_ocr | auto",
      "reasons": ["<reason_code>", ...],
      "artifact_paths": {"input": "...", "markdown": "...", "manifest": "..."},
      "priority": "high | medium | low"
    }

Reason codes and their meaning:

* `repetition_detected` — the sliding-window n-gram detector flagged
  hallucinated repetition in the output. **Priority: high.**
* `treatment_failed` — the compile exited non-zero. **Priority: high.**
* `source_page_provenance_incomplete` — the manifest is missing one or
  more source pages. **Priority: medium.**
* `materially_disagree_but_metrics_inconclusive` — the two backends produce
  structurally very different output but readiness is within ±3, so the
  automatic layer cannot rank them. **Priority: medium.**
* `auto_chose_uoc_but_tesseract_appears_stronger` — the auto selector
  picked UOC while Tesseract's structural counts substantially exceed
  UOC's. **Priority: medium.**
* `auto_chose_tesseract_but_uoc_appears_stronger` — mirror of the above.
  **Priority: medium.**
* `near_empty_output` — markdown length < 200 chars on a multi-page
  document (with a clean exit). **Priority: low.**

Reviewers add a `human_preference` field to the corresponding
`DocumentSummary` in `run_report.json` (either `"tesseract"` or
`"unlimited_ocr"`) to override the automatic label. The next report
regeneration picks this up via the schema round-trip helpers in
`benchmarks/ocr_auto_calibration/schema.py`.

## Canonical hallucination case status

As of the initial PR, no ParseBench document has a documented, reproducible
hallucination signature that we can pin as the canonical
cropped-rotated-scaled failure case. When such a case is confirmed:

1. Copy the PDF into `benchmarks/ocr_auto_calibration/fixtures/failure/`.
2. Add a sibling JSON with `profile_class` and a short `notes` field.
3. Re-run the harness — the failure loop will exercise it on all three
   treatments.

Until then, the harness runs the 12 ParseBench + 8 synthetic corpus; the
failure loop is a no-op.

## Constraints (what this PR intentionally does NOT do)

* **No production code changes.** The harness lives entirely under
  `benchmarks/ocr_auto_calibration/`, `tests/test_ocr_auto_calibration/`,
  and `docs/`. It does not modify `pdf.py`, `auto_selector.py`, the
  Unlimited-OCR backend, scoring, or the manifest schema.
* **No Auto Policy v1 threshold change.** The 3-page floor and 30%
  fraction stay exactly as merged in PR 100.
* **No policy recommendation without empirical data.** The Markdown
  report explicitly reports "undetermined" as the recommendation until an
  RTX 3060 pass has run.
